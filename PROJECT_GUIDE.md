# 项目实施建议书

## 项目概述

**题目**: 社交网络中的结构性隐私泄露风险：基于图拓扑匹配的去匿名化分析

**核心观点**: "即便我不说话，我的朋友也会暴露我"

## 一、推荐数据源与理由

根据你的项目需求和实现难度，我强烈推荐以下数据源优先级：

### 🥇 首选：GitHub（强烈推荐）

**优势**:
1. **API友好**: 官方REST API，文档完善，每小时5000次请求
2. **数据质量高**: 真实社交关系（Follow）+ 行为关系（Star/Fork）
3. **易于验证**: 公开数据，可交叉验证
4. **技术社区特征明显**: Rust/Go等小众社区形成明显的"结构指纹"
5. **学术认可度高**: 多篇顶会论文使用GitHub数据

**实施方案**:
```python
# 爬取步骤
1. 获取GitHub Personal Access Token (免费)
2. 搜索特定语言的活跃开发者（如Rust社区）
3. BFS爬取Follow关系，深度2-3层
4. 预期：1000-5000个节点，2-3小时完成
```

**特色实验**:
- **跨平台去匿名化**: GitHub上爬取的Follow关系 → 尝试在Twitter/LinkedIn上找回同一个人
- **异构图攻击**: 同时利用Follow关系 + Star关系构建更强的"指纹"

### 🥈 次选：Bilibili（适合展示）

**优势**:
1. **中文平台**: 更贴近国内评审老师
2. **共同关注UP主** = 强社区指纹
3. **可视化效果好**: 二次元、科技区等圈子边界清晰

**挑战**:
- 反爬虫较强，需要处理限流和验证码
- 需要登录cookies
- 爬取速度较慢

**实施方案**:
- 目标：爬取某个特定圈子（如虚拟主播圈、科技区）的用户关注网络
- 预期：500-1000个节点，1-2天完成

### 🥉 备选：SNAP公开数据集

如果时间紧张或爬虫遇到困难，直接使用：
- **Facebook Ego Networks** (4,039节点)
- **GitHub Social Network** (37,700节点)
- **Twitter Higgs Dataset** (456,626节点)

下载地址: https://snap.stanford.edu/data/

### ❌ 不推荐：微博

**原因**:
1. 反爬虫极强（IP封禁、验证码、登录验证）
2. API限流严格
3. 法律风险（《网络安全法》对爬虫有明确限制）
4. 数据获取成本高，时间长

## 三、预期实验结果（参考）

基于类似规模的图（1000-5000节点），预期结果：

| 方法 | 准确率 | Top-5 | Top-10 |
|------|--------|-------|--------|
| Baseline (传统特征) | 15-25% | 35-45% | 45-55% |
| DeepWalk (无种子) | 25-35% | 50-60% | 60-70% |
| DeepWalk + 5%种子 | 50-70% | 75-85% | 85-90% |

**关键发现**（预期）:
1. 即使删除30%的边，攻击准确率仍能达到50%+（有种子）
2. 仅需5%的种子节点，准确率就能从30%提升到60%+
3. 度中心性高的节点更容易被识别（"大V效应"）

## 五、避坑指南

### 技术坑
1. **图太大**: 控制在5000节点以内，否则DeepWalk训练很慢
2. **图太稀疏**: 确保平均度 > 3，否则特征不明显
3. **内存不足**: 使用稀疏矩阵，分批处理

### 数据坑
1. **爬虫被封**: 设置合理的delay，使用代理池
2. **数据质量差**: 过滤僵尸账号、孤立节点
3. **隐私合规**: 脱敏处理所有真实ID，仅用于学术研究

### 时间坑
1. **不要完美主义**: GraphSAGE可以作为"未来工作"
2. **优先核心功能**: 先跑通Baseline + DeepWalk
3. **提前准备答辩**: 最后3天只做PPT和演练
